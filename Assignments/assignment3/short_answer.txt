Const-correctness is the practice of using the const keyword to explicitly specify which variables, parameters, and member functions should not modify the state of an object. It includes:

Const variables: Variables that cannot be modified after initialization

Const parameters: Function parameters that promise not to modify the passed object

Const member functions: Methods that promise not to modify the object's state (don't change member variables)

Why it's important:

Compiler enforcement: The compiler catches accidental modifications at compile time

Self-documenting code: Makes the programmer's intent clear about what can/cannot be modified

Safer code: Prevents unintended side effects and bugs

Enables optimizations: Compiler can make better optimizations knowing certain values won't change

Better APIs: Clear contracts about what functions do and don't modify
